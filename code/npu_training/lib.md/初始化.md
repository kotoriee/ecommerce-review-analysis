计算任务环境初始化

创建计算任务容器：在您的项目中创建的计算任务在平台的算力中心被视为一个容器。不同算力资源的容器内有着不同的路径规则。
使用c2net库实现路径统一：为了简化用户体验，我们为您提供了名为c2net的Python库，该库的目标是在所有计算任务中实现路径的统一。
路径规范化的便利性：通过使用c2net库，用户无需修改其代码便可以轻松适配各种算力资源，从而实现对调试、训练任务容器中路径的统一管理。
获取路径
计算任务启动时后台将自动安装c2net库，若遇报错库不存在，请自行执行安装指令： pip install -U c2net

初始化
导入c2net包，并执行prepare()初始化
此步骤将返回一个c2net_context对象，该对象中的属性为各个数据资源的根目录
from c2net.context import prepare
c2net_context = prepare()
复制到剪切板错误复制成功
代码目录
c2net_context对象中的code_path属性为容器中的代码存放根目录
用户可将其储存为变量，方便后续代码中使用
在后续使用时，还需拼接计算任务所在的代码仓名称
codePath = c2net_context.code_path
codeFilePath = codePath + '/代码仓名称'.lower() + '/文件'
复制到剪切板错误复制成功
数据集目录
dataset_path属性为容器中的数据集根目录
注意此为数据集根目录，后续代码中加载数据集时还需拼接具体数据集名称及文件名称，具体用法可参考本页的数据集使用示例
datasetPath = c2net_context.dataset_path + '/数据集文件'
复制到剪切板错误复制成功
模型目录
pretrain_model_path属性为容器中的模型根目录
注意此为模型根目录，后续代码中加载模型时用户还需拼接具体模型名称及文件名称，具体用法可参考本页的模型使用示例
modelPath = c2net_context.pretrain_model_path + '/模型名称' + '/模型文件'
复制到剪切板错误复制成功
输出目录
output_path属性为容器中的输出目录，请将需要导出的数据存到此目录下
调试、训练任务中均可使用，保存后需执行 upload_output() 函数，会将容器内 output_path 路径下的文件回传到平台
from c2net.context import upload_output
outputPath = c2net_context.output_path
upload_output()
复制到剪切板错误复制成功

获取数据集文件路径
在代码中初始化，并拼接数据集名称与内部子路径以读取文件
# 导入包并执行初始化
from c2net.context import prepare
c2net_context = prepare()

# 获取数据集在容器内的根目录
datasetPath = c2net_context.dataset_path 

# 拼接 MnistDataset_torch 数据集训练文件 train-images-idx3-ubyte

from c2net.context import prepare

modelPath = c2net_context.pretrain_model_path 

# 拼接 chatglm3-6b 模型文件夹
chatglm_model_path = modelPath + '/chatglm3-6b'

# 拼接 Torch_MNIST_Example_Model 模型文件夹及 mnist_epoch1_0.73.pkl 模型文件

import argparse
parser = argparse.ArgumentParser()
args, unknown = parser.parse_known_args()
# 在这里，args 包含已知参数，unknown包含未知参数
