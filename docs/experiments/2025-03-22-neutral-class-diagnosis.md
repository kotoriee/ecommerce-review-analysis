# 中性类低准确率根因分析

## 日期
2025-03-22

## 问题
S2-S5 阶段中性类准确率持续低于 35-38%，远低于负面(84-88%)和正面(70-74%)。

---

## 数据分析结果

### train_3cls.json 软标签分布

| 类别 | 样本数 | 平均置信度 | 分布特征 |
|------|--------|-----------|---------|
| 负面 (0) | 1,952 | **0.900** | 99.3% ≥ 0.80 |
| 中性 (1) | 2,422 | **0.649** | 74.9% 在 [0.65, 0.70) |
| 正面 (2) | 1,659 | **0.906** | 97.7% ≥ 0.80 |

### 中性软标签详细分布

```
[0.50-0.55):    2  ( 0.1%)
[0.55-0.60):   14  ( 0.6%)
[0.60-0.65):  354  (14.6%)  ← 低置信边界样本
[0.65-0.70): 1814  (74.9%)  ← 高度集中，信号弱
[0.70-0.75):  195  ( 8.1%)
[0.75-0.80):   23  ( 0.9%)
[0.80-1.00):   20  ( 0.8%)
```

### curriculum train_2400.json

- 均衡采样：负面/中性/正面各 **800** 条
- 对话格式中的 confidence 字段：**100% = 0.80**（三类统一）
- **软标签未编码进训练数据**，仅使用硬标签 (0/1/2)

---

## 根因分析

### 根因 1：信号强度不对称（最核心）

```
负面/正面: soft_labels[主类] ≈ 0.90  →  信噪比 ~9:1
中性:      soft_labels[1]    ≈ 0.65  →  信噪比 ~2:1
```

标注时提示模板对混合评论的中性概率上限设为 0.60-0.70（由 `run_3cls_annotation.py` 的 prompt 决定），导致中性类的训练信号天然弱于正负两类。

### 根因 2：训练格式丢失软标签信息

`train_2400.json` 使用对话格式，assistant 输出为：
```json
{"sentiment": 1, "confidence": 0.80, "rationale": "..."}
```
所有样本 confidence 统一为 0.80，**软标签的差异信息未传入训练**。模型只能学到硬标签边界，无法利用置信度梯度。

### 根因 3：中性类方差极低（过度聚合）

```
中性 soft_labels[1]: mean=0.649, stdev=0.033
负面 soft_labels[0]: mean=0.900, stdev=0.021
```

74.9% 的中性样本聚集在同一概率区间 [0.65, 0.70)，模型学不到"程度不同的混合性"之间的区别。

### 根因 4：低置信边界样本污染（次要）

370 条中性样本（15.3%）的 soft_labels[1] < 0.65，其中多数偏向负面：
- 负面倾向中性：如 "Bought a set this week but the 'furry' part is already coming out in patches"
- 这类样本在评估时极易被模型分类为负面

---

## 影响评估

| 因素 | 影响程度 | 可优化性 |
|------|---------|---------|
| 信号强度不对称 | **高** | 中（需修改标注策略） |
| 训练格式丢失软标签 | **高** | **高**（可立即改） |
| 方差过低 | **中** | 中（需数据增强） |
| 边界样本污染 | **低** | 高（可过滤/重标注） |

---

## 优化建议

### 短期（立即可做）
1. **过滤边界样本**：移除 soft_labels[1] < 0.63 的中性样本（约 200 条）
2. **调整 loss 权重**：中性类 class_weight=2.0，加大梯度贡献

### 中期（需实验验证）
3. **提升标注置信度上限**：修改 `run_3cls_annotation.py` 中中性概率范围为 0.70-0.80
4. **在训练格式中编码软标签**：将 soft_labels 写入 assistant 响应或用 KL 散度 loss

### 长期
5. **重新标注中性样本**：增加标注细度（弱中性/强中性），提高类内区分度
