## 推理方案对比

| 方案 | 速度 | 显存占用 | 适用场景 | 复杂度 |
|------|------|---------|---------|--------|
| **Unsloth 单条** | ~0.1条/秒 (10s/it) | ~4.5GB | 开发调试 | 低 |
| **Unsloth 批量** | ~1-5条/秒 | ~6GB | 小批量测试 | 中 |

| **Ollama/GGUF** | 10-30条/秒 | ~4GB | 本地服务/API | 低 |
| **合并模型+vLLM** | 50+条/秒 | ~8GB | 生产部署 | 高 |

## 当前瓶颈分析

1. **单条 generate() 调用**: 每次都要重新准备 input tensors
2. **KV Cache 未预热**: 每条独立计算，无法复用
3. **Python 循环开销**: tqdm + 单线程处理
4. **模型加载方式**: base + LoRA 动态合并有 overhead

### 学习曲线趋势

| 阶段 | 数据量 | 准确率 | 中性准确率 | 趋势 |
|------|--------|--------|-----------|------|
| S1 | 600 | **64.00%** | — (200条样本) | 基准 |
| S2 | 1200 | **62.00%** | 33.8% | ↓ 下降 |
| S3 | 2400 | **60.40%** | 30.8% | ↓ 继续下降 |
| S5 | 8500 | **61.80%** | 37.8% | ↑ 小幅回升 |

**关键发现**:
1. **数据量增加并未提升准确率**，S1 (600条) 是目前最高，S3 是最低谷
2. **中性类识别是主要瓶颈** (30-38%)，严重拖累整体准确率
3. **负面识别稳定最好** (84-88%)，正面次之 (70-74%)
4. **S3→S5 出现小幅回升**：中性从 30.8% 升至 37.8%，提示大量数据有助于中性类
5. **问题假设**: 软标签噪声集中在中性边界样本，模型学到了偏置

**配置**:
- Batch size: 16
- Samples: 500
- Batches: 32
- 硬件: RTX 3070Ti 8GB

**实测速度**:
```
~25秒/批次 (16条)
预计总时间: ~13分钟 (32批次)
速度: ~0.6条/秒 (40条/分钟)
```

**对比**:
| 方法 | 速度 | 500条耗时 |
|------|------|-----------|
| Unsloth 单条 | 0.11条/秒 | ~75分钟 |
| **Unsloth 批量 (bs=16)** | **~0.6条/秒** | **~13分钟** |
| Ollama/GGUF (预计) | ~25条/秒 | ~20秒 |

**瓶颈分析**:
- 25秒/批次过慢，理论应在 5-8秒/批次
- 可能是 WSL I/O 或 tokenization 开销
- 后续改用 Ollama/GGUF 方案 (~25条/秒)

**结论**: Unsloth 批量比单条快 6x，但仍不够快。Ollama/GGUF 是最佳选择。
